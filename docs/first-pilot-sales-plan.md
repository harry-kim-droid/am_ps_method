# First Pilot Sales Plan

## Goal

Get one paid or high-intent pilot within 7 days without increasing API spend.

Success means one team agrees to:

- Let you review 3 recent pull requests
- Share whether the findings are useful
- Consider a 2-week paid pilot if the sample has value

Do not promise guaranteed savings or autonomous engineering. Sell a human-verified review service that uses Codex to speed up the first pass.

## Best First Customers

Prioritize teams where trust is already warm:

- A founder or developer you already know
- A freelancer or small agency maintaining client code
- A SaaS team with 2-10 engineers
- A team using GitHub PRs but without a dedicated senior reviewer
- A team that recently hired or is hiring backend/full-stack engineers

Avoid the first pilot with:

- Banks, hospitals, or highly regulated data
- Teams demanding production credentials
- Teams expecting guaranteed bug prevention
- Teams that want automatic code changes before seeing review quality

## 7-Day Plan

Day 1:

- Build a list of 30 warm or semi-warm prospects.
- Put them in `templates/prospect-tracker.csv`.
- Send 10 short DMs.

Day 2:

- Send 10 more DMs.
- Follow up with anyone who replied.
- Offer a free sample review of 3 recent PRs.

Day 3:

- Send 10 more DMs.
- Convert interested replies into a 20-minute call.
- Use `docs/pilot-proposal.md` after the call.

Day 4:

- Perform one sample review for the strongest prospect.
- Track every useful finding, weak finding, and false positive.

Day 5:

- Send the sample report.
- Ask which findings were useful enough to act on.

Day 6:

- Follow up with non-responders.
- Ask one direct question: "Would this be useful if it appeared on every important PR?"

Day 7:

- Offer the 2-week pilot.
- Use a simple close: KRW 300,000 for 1 repo, selected PRs, and 1 weekly report.

## Minimum Viable Offer

Use this offer until one customer pays:

- 2 weeks
- 1 repository
- Up to 20 PRs
- 1 weekly technical debt report
- Human-verified findings only
- KRW 300,000, or free only if the customer is strategically valuable

## Qualification Questions

Ask these before setup:

- How many PRs do you merge each week?
- Who reviews risky backend or data changes?
- What kind of bug is most expensive when it slips through?
- Do you prefer draft reports first or direct PR comments?
- Would a weekly ranked cleanup list help sprint planning?

## Conversion Rule

Do not convert based on praise. Convert only when the customer says at least one of:

- "This would have saved review time."
- "We should fix this."
- "This is the kind of issue we miss."
- "Can this run on our next PR?"

## Stop-Loss Rule

Stop or re-scope the pilot if:

- Setup takes more than 2 hours
- The customer wants broad consulting outside PR review
- The customer cannot name a repository or decision maker
- The first 3 sample PRs produce no useful findings

## Daily Metrics

Track:

- Prospects contacted
- Replies
- Sample review offers accepted
- Calls booked
- Sample findings sent
- Paid pilot asks made
- Paid pilots closed

Use these numbers to decide whether the offer, audience, or message needs to change.
