# Unit Economics

## Cost Model

Track each customer with these numbers:

| Metric | Starter Target | Team Target |
| --- | ---: | ---: |
| Repositories | 1-2 | 3-5 |
| PRs/month | 50 | 150 |
| Weekly reports/month | 4 | 4 |
| Human review time/PR | 5-10 min | 3-8 min |
| Human report time/week | 30-60 min | 45-90 min |
| Monthly price | KRW 300,000-700,000 | KRW 900,000-2,000,000 |

## Margin Rule

Keep direct operator time below:

- Starter: 4 hours/month
- Team: 10 hours/month

If a customer needs more time, either:

- raise the monthly plan
- reduce scope
- automate a repeated step
- end the pilot politely

## Pricing Logic

The customer should feel the price is less than:

- a few hours of senior engineer time
- one avoided production bug
- one avoided week of messy cleanup
- one extra contractor review cycle

## Upgrade Triggers

Move a customer to a higher plan when:

- they add repositories
- they want automatic PR comments
- they want fix PRs
- they want security-focused review
- they want issue creation and sprint planning support

## Red Flags

Avoid customers who:

- expect fully autonomous code changes from day one
- cannot define what useful review means
- want production credentials shared casually
- measure success only by volume of comments
- reject human verification but blame you for AI noise
